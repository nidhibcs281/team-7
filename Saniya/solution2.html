Fraud Detection in Private Banks

SDG: 11

Target: 11.2

Indicator: 11.2.1

An Income Tax Officer is tasked with auditing four banks (Bank A, Bank B, Bank C, and Bank D). The officer needs to minimize the total distance traveled while visiting all the banks starting from Bank A. Additionally, there is a need to ensure the security and integrity of banking transactions by detecting fraudulent activities at each bank.

The banks and their interconnections are represented as a graph:

  • Each bank is a vertex (Bank A = 0, Bank B = 1, Bank C = 2, Bank D = 3).
  • The roads connecting the banks have specific distances (weights).
  • The graph is implemented using an adjacency list.

Algorithm 1: Dijkstra's Algorithm

Description: Computes the shortest path from Bank A to all other banks.

Efficiency: O(|E| log|V|)

Algorithm 2: Brute Force Search Algorithm

Description: Identifies fraudulent transactions in the transaction records of each bank using a predefined fraud pattern (e.g., transactions containing the keyword "Fraudulent").
